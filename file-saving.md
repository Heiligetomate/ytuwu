# File Saving Process

To show this process, I will show an example of the saving of an entire channel.

- A channel has a vec with singles and two vecs with playlists in them
- Every playlist has a vec with media objects in them (same as the singles)
- Every media can have one or n streams in them
- Every media can have one or zero thumbnails in them

## Streams and Thumbnails

The main saving function for streams is [save_media_stream()](src/downloader/streams/util.rs). It takes a path and a file name. The path to the file is generated by taking the given path and adding the file name. The ending is automatically added.

This function gets used by every struct implementing the [MediaStream](src/downloader/streams/core.rs) trait. An example of the [save()](src/downloader/streams/core.rs) function can be found [here](src/downloader/streams/audio.rs). As you can see, it is just a wrapper for [save_media_stream()](src/downloader/streams/util.rs).

A [Thumbnail](src/downloader/thumbnail.rs) is basically structured like a media stream. It has the pure bytes in it and a file name. The [save()](src/downloader/thumbnail.rs) function takes a path and adds the file name to it. The extension is always `.png`.

## Downloaded Media

There are two different types of downloaded media: media with one stream and media with n streams. Both types can have a thumbnail and both types have the same metadata.

Both types have three saving methods:

- [save_thumbnail()](src/downloader/media/downloaded.rs) — a wrapper for the thumbnail save discussed above
- [DwnMedia::save_media_stream()](src/downloader/media/downloaded.rs) — a wrapper for saving a single stream
- [DwnBundleMedia::save_media_streams()](src/downloader/media/downloaded.rs) — saves all streams and appends an index to the file name if there are duplicate extensions (e.g. `x.webm`, `x.mp3` but `y.webm`, `y-1.webm`)
- [save_full()](src/downloader/media/downloaded.rs) — calls both of the functions discussed above

## Downloaded Playlists

[DwnList](src/downloader/playlist/downloaded.rs) has a vec of [DwnMedia](src/downloader/media/downloaded.rs).
[DwnBundleList](src/downloader/playlist/downloaded.rs) has a vec of [DwnBundleMedia](src/downloader/media/downloaded.rs).

Both have two saving methods:

- [DwnList::save()](src/downloader/playlist/downloaded.rs) — calls [DwnMedia::save_media_stream()](src/downloader/media/downloaded.rs) on each media object
- [DwnBundleList::save()](src/downloader/playlist/downloaded.rs) — calls [DwnBundleMedia::save_media_streams()](src/downloader/media/downloaded.rs) on each media object
- [save_with_dir()](src/downloader/playlist/downloaded.rs) — creates an additional directory in the given path and calls [save()](src/downloader/media/downloaded.rs) with the new path

## Downloaded Channel

[DwnChannel](src/downloader/channel/downloaded.rs) has a vec of [DwnList](src/downloader/playlist/downloaded.rs).
[DwnBundleChannel](src/downloader/channel/downloaded.rs) has a vec of [DwnBundleList](src/downloader/playlist/downloaded.rs).

First, three folders are created:

- `path/singles/`
- `path/eps/`
- `path/albums/`

After that, a save function is called which does the following:

- Calls [DwnMedia::save_media_stream()](src/downloader/media/downloaded.rs) / [DwnBundleMedia::save_media_streams()](src/downloader/media/downloaded.rs) for each single
- Calls [save_with_dir()](src/downloader/playlist/downloaded.rs) for each EP or album
